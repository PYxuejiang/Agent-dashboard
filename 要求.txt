最可能、最省事、也最稳的技术路线，不是让 PyCharm 直接做，而是：

## 你现在这套 logger 怎样复用最好

你有两种接法。

### 接法 A：直接解析 jsonl

最简单、最稳。

优点：

 不依赖 MCP 会话
 不依赖 Agent 是否在线
 小组件只读磁盘
 实时性足够

逻辑就是：

1. 扫描 `Dai_journalstore`
2. 找今天日期目录
3. 读取各项目下的 `.jsonl`
4. 提取：

    最新 `close_step.summary`
    最新 `issue`
    最新 `prompt.abstract_task`
    最新 `next_action`
5. 聚合显示

这是我最推荐的首版实现方式。

---

### 接法 B：桌面组件直接 import `mcp_logger.py`

也能做。

优点：

 能复用你已经写好的 `list_projects`  `get_project_overview`  `search_project_logs`
 逻辑更统一

缺点：

 这些函数原本是 MCP tool 风格，返回 JSON 字符串
 你还得处理 `ROOT`、状态和调用边界
 不如直接读 jsonl 直观

所以我的建议是：

首版桌面组件：直接读 jsonl
第二版：再考虑复用 `mcp_logger.py` 的查询函数

## 这个小组件内部的数据结构应该怎么设计

建议在桌面程序里做一个聚合层，把原始事件转成“今天概览”。

例如：

```json
{
  date 2026-04-14,
  projects [
    {
      project PTM_GNN_SEQ,
      event_count 8,
      chat_count 2,
      last_summary 已完成 fusion stageB w63 后台命令整理，并统一日志输出路径,
      last_next_action 继续比较 stageB w63 与 stageA 指标差异,
      open_issue_count 1,
      solved_issue_count 2,
      latest_chat fusion_stageB_w63,
      files {
        md ...fusion_stageB_w63.md,
        jsonl ...fusion_stageB_w63.jsonl
      }
    }
  ]
}
```

这样前端显示会很容易。
