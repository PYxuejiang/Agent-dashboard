【2】拖拽移动实现
────────────────────────────────────────────────────────────────────────────

技术点：鼠标事件捕获 + 全局坐标计算

实现：
```python
def mousePressEvent(self, event):
    if event.button() == Qt.LeftButton:
        # 记录鼠标相对窗口的偏移
        self._drag_pos = event.globalPosition().toPoint() - self.frameGeometry().topLeft()

def mouseMoveEvent(self, event):
    if self._drag_pos and event.buttons() == Qt.LeftButton:
        # 根据偏移移动窗口
        self.move(event.globalPosition().toPoint() - self._drag_pos)

def mouseReleaseEvent(self, event):
    self._drag_pos = None
```

关键：
- globalPosition() 获取屏幕绝对坐标
- frameGeometry().topLeft() 获取窗口左上角坐标
- 偏移量 = 鼠标全局坐标 - 窗口坐标


【3】右下角拖拽缩放
────────────────────────────────────────────────────────────────────────────

技术点：热区检测 + 动态 resize

实现：
```python
_RESIZE_MARGIN = 8  # 热区大小

def _in_resize_zone(self, pos: QPoint) -> bool:
    # 判断鼠标是否在右下角 16x16 区域
    return (pos.x() >= self.width() - _RESIZE_MARGIN * 2 and
            pos.y() >= self.height() - _RESIZE_MARGIN * 2)

def mousePressEvent(self, event):
    if self._in_resize_zone(event.position().toPoint()):
        self._resizing = True
        self._resize_start = event.globalPosition().toPoint()
        self._resize_size = QSize(self.width(), self.height())

def mouseMoveEvent(self, event):
    if self._resizing:
        delta = event.globalPosition().toPoint() - self._resize_start
        self.resize(
            max(250, self._resize_size.width() + delta.x()),
            max(200, self._resize_size.height() + delta.y())
        )
```

优化：
- 限制最小尺寸防止窗口过小
- 鼠标进入热区时显示 SizeFDiagCursor 光标提示
- 使用 setMouseTracking(True) 实时更新光标


【4】小悬浮图标
────────────────────────────────────────────────────────────────────────────

技术点：独立 QWidget + 点击/拖拽区分

实现：
```python
class FloatIcon(QWidget):
    def __init__(self, panel: "MainWidget"):
        self.panel = panel
        self.setFixedSize(48, 48)
        lbl = QLabel("📓", self)  # 使用 emoji 作为图标

    def mouseReleaseEvent(self, event):
        # 判断是点击还是拖拽
        moved = (event.globalPosition().toPoint() - self.frameGeometry().topLeft()) != self._drag_pos
        if not moved:
            # 点击：切换面板显示
            if self.panel.isVisible():
                self.panel.hide()
            else:
                # 面板出现在图标旁边
                geo = self.frameGeometry()
                self.panel.move(geo.right() + 8, geo.top())
                self.panel.show()
```

关键：
- 独立窗口，与主面板解耦
- 通过位置变化判断是点击还是拖拽
- 面板位置根据图标位置动态计算

def _read_jsonl(path: Path) -> List[Dict]:
    events = []
    with open(path, "r", encoding="utf-8") as f:
        for line in f:
            line = line.strip()
            if line:
                events.append(json.loads(line))
    return events

# 提取关键信息
for ev in events:

═══════════════════════════════════════════════════════════════════════════════

【数据流】
1. MCP Agent 工作 → 调用 mcp_logger.py
2. mcp_logger.py → 写入 JSONL 到 Dai_journalstore/[项目]/[日期]/[聊天].jsonl

   - 启动文件监听
   - 启动定时器

   - 文件变化 → 自动刷新
   - 定时器触发 → 定期刷新
   - 用户交互 → 切换视图、查看详情

3. 退出：
   - 停止文件监听
   - 停止定时器
   - 清理资源

═══════════════════════════════════════════════════════════════════════════════
五、关键技术难点与解决方案
═══════════════════════════════════════════════════════════════════════════════

【难点 1】无边框窗口的白边问题
原因：Qt 默认窗口背景为白色，透明属性未完全生效
解决：
- 设置 WA_TranslucentBackground
- 显式设置 background: transparent
- 外层布局边距设为 0

【难点 2】拖拽与点击的区分
原因：鼠标按下和释放位置可能有微小偏移
解决：
- 记录按下时的位置
- 释放时比较位置差异
- 差异小于阈值视为点击

【难点 3】右下角缩放与拖拽冲突
原因：两者都监听鼠标事件
解决：
- 优先判断是否在缩放热区
- 使用状态标志区分当前模式
- mouseMoveEvent 中根据状态分别处理

【难点 4】多日期数据聚合效率
原因：7天数据需要遍历多个目录和文件
解决：
- 使用 Counter 高效统计
- 只提取必要字段
- 限制返回数量（Top N）

【难点 5】JSONL 格式兼容性
原因：不同版本 logger 可能字段不同
解决：
- 使用 .get() 而非直接访问
- 同时检查 payload 和顶层字段
- 提供默认值

═══════════════════════════════════════════════════════════════════════════════
十一、技术选型理由
═══════════════════════════════════════════════════════════════════════════════

【为什么选 PySide6？】
- 跨平台，一套代码多端运行
- 成熟稳定，Qt 生态完善
- Python 原生，开发效率高
- 支持复杂 UI，可定制性强

【为什么选 JSONL？】
- 支持追加写入，无需重写整个文件
- 逐行解析，内存占用低
- 结构化数据，易于查询
- 人类可读，便于调试

【为什么选 watchdog？】
- 跨平台文件监听
- 事件驱动，无需轮询
- 性能优秀，资源占用低
- API 简洁，易于集成

【为什么不用 Electron？】
- Python 生态更适合数据处理
- 内存占用更低
- 启动速度更快
- 无需打包 Chromium

本项目综合运用了以下核心技术：
1. PySide6 GUI 编程（无边框窗口、信号槽、样式系统）
2. 文件系统监听（watchdog）
3. JSONL 数据解析与聚合
4. 多视图切换（StackedWidget）
5. 自定义拖拽与缩放
6. 跨平台兼容性处理
7. 配置管理与持久化
8. 性能优化（懒加载、增量更新）

通过模块化设计和松耦合架构，实现了一个高效、易用、可扩展的桌面日志监控工具。
